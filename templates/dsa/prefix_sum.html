Prefix Sum

A simple technique that allows for fast calculation of sums of elements in a given slice(contiguous segments of array). The idea of prefix sums is defined as the consecutive totals of the first 0, 1, 2......n elements of an array.

We can easily calculate the prefix sums in O(n) time complexity. Notice that the total pk equals pk-1 + ak-1, so each consecutive value can be calculated in a constant time. Make a section to hold the algorithm and code finding prefix sums.

The suffix sums can also be calculated which are the totals of the k last values.

When asked to find the sums of m slices[x..y] between x and y where 0 ≤ x ≤ y < n, the simplest approach of iterating over each element to find the sums is by using a nested approach. The time complexity for a nested approach will be O(n * m) complexity. However, when using the prefix or the suffix sums, The prefix sums for the entire array can be calculated and the target sum can be obtained in constant time O(1).

Hence, using the above table, the sum between two indices x and y in any given array can be obtained in constant time.
